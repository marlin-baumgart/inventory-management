Factory Inventory Management System

A full-stack demo application for inventory tracking, order management, demand forecasting, and spending analytics. Vue 3 single-page app talking to a FastAPI service over JSON, with all data held in memory from JSON files.

Data Flow

  1. User changes a filter in the UI
    The FilterBar component updates shared state in the useFilters composable — refs like selectedWarehouse, selectedCategory, selectedStatus, selectedMonth.
  2. View triggers an API call
    A watcher on filter state calls api.getOrders(filters) (or similar) from client/src/api.js, which serializes filters into query params and skips any value of 'all'.
  3. FastAPI receives the request
    Endpoint in server/main.py accepts optional warehouse, category, status, month query params. The month param accepts either YYYY-MM or quarter syntax (Q1-2025–Q4-2025).
  4. In-memory filtering
    Helpers apply_filters() and filter_by_month() filter the global lists loaded from server/data/*.json. Filtering happens on copies — original lists are never mutated.
  5. Pydantic validates the response
    Each endpoint declares a response_model (e.g. List[Order]). Pydantic validates the shape on the way out and FastAPI serializes to JSON.
  6. Vue stores raw data, derives the rest
    The view component stores the response in a ref (e.g. allOrders) and exposes derived values through computed properties — totals, groupings, chart series. The template re-renders reactively.
